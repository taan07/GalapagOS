# Chunk 1 — Manager chat spine (Darwin)

Chunk 1 is implemented by Claude as the reference-quality standard for later
chunks. Read `docs/vision.md` and `docs/architecture.md` first — they are the
binding contract and override this brief on any conflict. Consult the Claude
Agent SDK docs (code.claude.com/docs/en/agent-sdk) — do not guess SDK APIs.

## State when you start

Fresh repo: docs, package.json (zod + tsx included), tsconfig, tsconfig.node.json
committed. `npm install` not yet run.

## Build

- `src/config.ts` — env with defaults: `GALAPAGOS_VAULT_PATH`
  (`/Users/taan/Documents/Obsidian Vault`), `GALAPAGOS_MANAGER_MODEL`
  (`claude-fable-5`), `GALAPAGOS_DAEMON_PORT` (4517), `GALAPAGOS_STATE_DIR`
  (`~/.galapagos`). No target-root env — projects are registered rows.
- SQLite spine (`src/adapters/db/`): `schema.sql` + WAL + migrate-on-open at
  `<state-dir>/state.db`; tables `projects`, `manager_sessions`,
  `manager_turns` (with `sdk_session_id_after`, `distilled_at`), `jobs`.
  Repos: `projects.ts` (register — validate path, offer `initGit` for non-git
  projects via a narrowly-scoped mutating git runner — list, get),
  `manager.ts` (sessions/turns).
- Git truth (ported from the prior prototype, read-only):
  `src/core/git/parsers.ts` + `src/adapters/git/runner.ts` + their tests.
- Vault specifics (`src/adapters/vault/specifics.ts`): write/list agreed
  specifics per architecture §1 "Interim memory" — wx creates, inside-vault
  guard, frontmatter shaped for Chunk 2 ingestion.
- Manager tools (`createSdkMcpServer`): `git_truth(scope)`,
  `record_specific(question, answer)`, `list_specifics()`, `read_records`
  honest stub ("records store arrives in Chunk 2" — never fake data).
- Manager session (`src/adapters/agent/manager-session.ts`): per-project
  `query()` with resume + cwd (= project root, ALWAYS) + pinned model +
  doctrine + galapagos MCP server + allowedTools (galapagos tools,
  Read/Glob/Grep) + deny-fallback `canUseTool`; persist session id after every
  turn; on resume failure start fresh with a **visible** "re-briefed" event —
  never silently blank.
- Doctrine (`src/daemon/doctrine.ts`): Darwin — relentless specifics per
  vision.md; observed-vs-claimed discipline (git_truth before asserting git
  facts); consult list_specifics before proposing; record agreed answers via
  record_specific immediately; Chunk 1 boundaries (no file edits, no workers).
- Daemon (`src/daemon/main.ts`) on :4517: `GET /health`,
  `POST /manager/message` (SSE reply), `GET /events`. No Express. No
  state-bearing module memory beyond live SSE clients.
- Next UI: project picker + add-project form (with git-init offer); chat page
  (history from SQLite, queued input while Darwin works, streamed replies,
  subtle inline expandable tool chips); always-visible "Agreed specifics"
  panel reading the vault; dark calm-mission-control CSS (no Tailwind);
  explicit error states (dead daemon = visible banner). UI never imports
  adapters — route handlers only.
- Resume spike: `npm run spike:resume` (opt-in, real SDK call): turn → capture
  id → new wrapper → resume → assert recall; assert cwd-mismatch produces the
  visible re-brief path.

## Forbidden

No workers, no lanes, no monitoring, no records store (vault specifics +
honest stub only), no Convex/cloud, no API keys (subscription auth via local
Claude Code login), no decorative dashboard filler.

## Exit criterion

Register a real project through the picker; hold a multi-turn conversation
with Darwin about its actual git state (git_truth chip visible); give him a
lazy feature request and get properly interrogated, with agreed answers
appearing in the Obsidian vault and the specifics panel; kill daemon + Next;
restart; the conversation continues (or visibly re-briefs).

## Verification

- `npm test` green (typecheck + parsers/adapter/db/specifics/tools suites).
- Manual drills: pester drill, restart drill, project-switch drill.
- `npm run spike:resume` passes.
- `sqlite3 ~/.galapagos/state.db 'select count(*) from manager_turns'` grows
  per exchange.
