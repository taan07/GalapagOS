You are implementing Chunk 1 of Galapagos. Read `docs/vision.md` and
`docs/architecture.md` first — they are the binding contract and override this
brief on any conflict. Consult the Claude Agent SDK docs (docs.claude.com,

do not guess SDK APIs.

## State when you start

Fresh repo: docs, package.json, tsconfig, .gitignore committed. No src/ code

## Build

- SQLite bootstrap (`src/adapters/db/`): `schema.sql` + db open/migrate helper,
  creating `projects`, `manager_sessions`, `manager_turns`, `jobs` (columns per
  architecture §3). DB path: `.galapagos/state.db` inside the target project
  root (the target project is configured by env `GALAPAGOS_TARGET_ROOT`,
  defaulting to this repo itself for development).
- Daemon (`src/daemon/main.ts`) on localhost:4517: `GET /health`,
  `POST /manager/message` streaming the reply as SSE, `GET /events` (SSE,
  minimal: broadcast new manager turns).
- Manager session wrapper (`src/adapters/agent/manager-session.ts`): one
  `query()` per user turn with `resume: lastSdkSessionId`; persist the
  post-turn session id to `manager_turns.sdk_session_id_after` on every turn;
  persist every user/assistant turn to `manager_turns` as it streams.
- Manager tools (`src/adapters/agent/manager-tools.ts` via
  `createSdkMcpServer`): `git_truth(scope)` backed by the ported read-only git
  runner + parsers; `read_records` as an honest stub returning "records store
  arrives in Chunk 2" (never fake data).
- Ports (architecture §10): `core/git/parsers.ts` and `adapters/git/runner.ts`
  from the prior prototype, verbatim, with their tests
  (`/Users/taan/Dev/karz98rk/src/git-truth/parsers.ts`, `.../adapter.ts`,
  `/Users/taan/Dev/karz98rk/tests/parsers.test.ts`, `.../tests/adapter.test.ts`).
- Chat UI: Next.js page at `/` — message list + input, streaming manager
  replies, calm-mission-control styling per vision (quiet, whitespace, no
  decorative noise). Next route handlers proxy to the daemon; UI never imports
  adapters.
- Resume spike test: an integration test that writes a turn, simulates process
  restart (new session wrapper instance), and proves the conversation continues
  via `resume` — or falls back visibly.
- Dev scripts: `npm run dev` starts daemon + Next together; `npm test` =
  typecheck + node --test.

## Forbidden

No workers, no lanes, no records store (stub only), no monitoring loop, no
distillation, no Convex or any cloud service, no API keys (the Agent SDK uses
the local subscription-authenticated Claude), no decorative dashboard filler.

## Exit criterion

A user holds a multi-turn conversation with the manager about the target repo's
real git state (via `git_truth`), kills both daemon and Next, restarts, and the
conversation continues — or visibly re-briefs, never silently starts blank.
